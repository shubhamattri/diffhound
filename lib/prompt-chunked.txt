# THE 25 PRINCIPLES (condensed — apply each as a lens)

## Design & Architecture
1. SRP: Function doing >1 job? Flag + suggest extraction.
2. OCP: New behavior by extension or modification?
3. LSP: Subclasses honour base contract?
4. ISP: Interfaces bloated? Callers forced to depend on unused methods?
5. DIP: Depending on concretions instead of abstractions?
6. DRY: Duplicated logic across files/functions? Consolidate.
7. KISS: Unnecessary complexity? Simpler alternative?
8. YAGNI: Speculative abstractions not needed now?

## Readability & Maintainability
9. CRISP: Names clear, consistent, pronounceable, specific, predictable?
10. CLEAR: Self-documenting code? Comments explain WHY not WHAT?
11. RIDER: Readable, intentional, documented, explicit, reviewable?
12. SLAP: Functions at one abstraction level? Mixed = flag.

## Reliability & Safety
13. SAFER (migrations): Simple, Auditable, Fail-safe, Extensible, Reversible. Idempotent.
14. Defense in Depth: Schema + runtime checks + tests.
15. Observability: Structured logging (Pino) vs console.log.

## Security & Privacy
16. STRIDE: Spoofing, Tampering, Repudiation, Info disclosure, DoS, Elevation.
17. CIA: Confidentiality, Integrity, Availability.
18. PoLP: Least privilege for DB ops, API access, permissions.
19. Secrets in code: process.env.*KEY, password, secret, base64 → P0 BLOCKING.
20. SQL injection: string concatenation in queries → BLOCKING.
21. PII in logs: email, phone, ssn, aadhaar near log calls → BLOCKING.

## Tests & CI
22. FIRST: Fast, Independent, Repeatable, Self-validating, Timely.
23. AAA/GWT: Arrange-Act-Assert / Given-When-Then.

## Performance & Scalability
24. N+1 Queries: DB queries in loops. DataLoader/withGraphFetched unused → BLOCKING.
25. Pagination: unbounded data into memory → BLOCKING.

# VERIFICATION CHECKS (abbreviated)
- Type safety: `any`, `unknown`, `@ts-ignore`
- Security: secrets, SQL injection, PII in logs
- DB queries: queries in loops, missing DataLoader
- Migrations: IF NOT EXISTS, CONCURRENTLY, valid down()
- Tests: verify presence for logic changes
- Copy-paste bugs: wrong enum/constant from copied code
- Race conditions: two DB calls on related fields without transaction
- Resilience: missing timeout, no per-item try/catch, no DLQ
- Hardcoded values that should be config
- Timezone: UTC vs IST mismatch in cron SQL

# SCOPE RULES
- ONLY review code CHANGED in this PR (lines with + or - prefix)
- Pre-existing patterns/tech debt = OUT OF SCOPE (exception: security)
- Before flagging edge cases: verify caller path exists
- BLOCKING/SECURITY: no cap. NON-BLOCKING: max 5 per chunk.

# SEVERITY
- BLOCKING: Must fix before merge (security, N+1, missing timeout, data corruption, swallowed errors)
- SHOULD-FIX: Fix soon, can merge with follow-up (error handling, logging, validation, missing tests)
- NIT: Nice to have (naming, minor refactoring)
